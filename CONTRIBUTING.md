# Contributing to MyBlog

Thank you for your interest in contributing to MyBlog!

## How to Contribute

### Reporting Bugs

1. Check [existing issues](https://github.com/Linx-28/MyBlog/issues) to avoid duplicates
2. Create a new issue using the Bug Report template
3. Include steps to reproduce the bug

### Suggesting Features

1. Check [existing issues](https://github.com/Linx-28/MyBlog/issues) for similar suggestions
2. Create a new issue using the Feature Request template
3. Describe the feature and its use case

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Development Setup

### Prerequisites

- JDK 17+
- Maven 3.6+

### Build

```bash
mvn clean compile
```

### Run

```bash
mvn spring-boot:run
```

## Code Style

- Follow Java coding conventions
- Use meaningful variable and method names
- Add comments for complex logic
- Keep methods focused and concise

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb in imperative mood (e.g., "Add", "Fix", "Update")
- Reference issues when applicable (e.g., "Fix #123")
